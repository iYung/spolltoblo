# Disconnect / Rejoin Checklist

- [x] Task A — `src/utils/id.js` — Persist peerId to localStorage so it survives page reloads. Replace the current `generateId()` export with a `getOrCreateId()` function that reads `localStorage.getItem('spolltoblo-peer-id')` and returns it if present and non-empty, otherwise generates a new random ID with the existing logic, writes it to localStorage under that key, and returns it.

- [x] Task B — `server/index.js` — Implement soft disconnect, rejoin, and kick on the server. Three changes to the same file:
  1. **Soft disconnect**: In `ws.on('close')`, instead of deleting the peer and broadcasting `'peer-left'`, set `peer.disconnected = true` and set `peer.ws = null`, then broadcast `{ type: 'peer-disconnected', peerId: currentPeerId }` to all still-connected peers in the room. Only delete the room entry when `room.size === 0` (all slots gone) — but don't delete disconnected slots just because they have no ws. Clean up disconnected slots only when the room becomes fully empty (all remaining peers are also disconnected).
  2. **Rejoin handler**: Add a `'rejoin'` case in the message switch. The client sends `{ type: 'rejoin', roomId, peerId, name }`. Look up `rooms.get(roomId)?.get(peerId)`. If found and `disconnected === true`: set `peer.ws` to the new `ws`, clear `peer.disconnected`, update `currentRoom` and `currentPeerId` locals. Broadcast `{ type: 'peer-rejoined', peerId, name, joinOrder: peer.joinOrder }` to all other connected peers. Reply to the rejoining peer with `{ type: 'room-peers', peers: [...] }` containing all other peers in the room (same shape as the initial join response: `[{ peerId, name, joinOrder }]`). If the peerId is not found or not disconnected, fall through and treat as a normal `'join'`.
  3. **Kick handler**: Add a `'kick'` case. The client sends `{ type: 'kick', roomId, targetPeerId }`. Look up the target in the room. Only act if the target exists and `target.disconnected === true`. Delete the target from the room map. Broadcast `{ type: 'peer-left', peerId: targetPeerId }` to all remaining connected peers.

- [x] Task C — `src/components/Room.jsx` — Four changes to the client:
  1. **Use persisted identity**: Import `getOrCreateId` from `../utils/id` and use it to initialize `myId` (replace the `generateId()` call).
  2. **Rejoin on connect**: After opening the WebSocket and before sending `'join'`, check if `rooms.get(roomId)` might have the peer's slot — the client doesn't know, so always send `{ type: 'rejoin', roomId, peerId: myId.current, name: playerName }` instead of `'join'`. The server falls through to normal join if the slot isn't found. Remove the old `'join'` send.
  3. **Handle `'peer-disconnected'`**: In the message switch, add a case that closes and removes the peer's RTCPeerConnection (`pcsRef.current[msg.peerId]?.close()`, `delete pcsRef.current[msg.peerId]`), then calls `setPeers` to set `peers[msg.peerId].disconnected = true` (merge, do not delete the peer entry). Do NOT touch `gameState`.
  4. **Handle `'peer-rejoined'`**: Add a case that clears `peers[msg.peerId].disconnected` (set to false). Then re-establish WebRTC: create a new RTCPeerConnection for that peerId (same as the `'peer-joined'` path — add tracks, create offer, etc.). Also fire the existing game-state sync (broadcast own life, commanders, deck) so the rejoining player gets current state. Pass an `onKick` handler to `PlayerVideo` that sends `{ type: 'kick', roomId, targetPeerId }` over the WebSocket.

- [x] Task D — `src/components/PlayerVideo.jsx` — Render a disconnected overlay when `player.disconnected === true`. The overlay sits above the video element (absolutely positioned, full card size, semi-transparent dark background). It contains two elements: a "Disconnected" label and a "Kick" button. The Kick button is only rendered when an `onKick` prop is provided (i.e. it is not the local player's own card — Room.jsx should not pass `onKick` for the local player). Clicking Kick calls `onKick(player.peerId)`. The overlay should not be rendered at all when `player.disconnected` is falsy.
